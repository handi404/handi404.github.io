3.异常处理

本文最后更新于 2025年7月13日 下午

我们将实现一个异常处理或全局异常。
在我们的应用程序中有一个处理程序，它的作用是监听所有应用程序的控制器并捕获所有可能发生的异常类型。

当我们尝试登录用户或内部发生某些异常时，例如 AuthenticationService 中注册时未找到 name 为 USER 的 Role 抛出的异常，激活账户中抛出的异常等等。

我们有任何类型的异常，我们需要处理并最终向用户返回一个合适的响应，所以在这一部分中在我们的应用程序中处理异常。

[[Java/Spring/Spring Boot/Exception/自定义运行时异常|自定义运行时异常]]

自定义运行时异常

定义错误响应体

在 handler 包中创建 ExceptionResponse 类表示错误响应结构

使用枚举管理错误码

创建一个 enum，包含所有可能的业务错误代码和希望返回给用户的描述和 HTTP 状态。

在 handler 包中创建 BusinessErrorCodes enum

创建全局异常处理器

在 handler 包中创建 GlobalExceptionHandler 全局异常处理器，进行捕获并处理不同异常做出响应。

现在我们实现了异常处理，每当我们得到或一个异常发生，我的应用程序会处理它。
