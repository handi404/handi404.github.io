介绍 JPA Auditing(审计)，该功能可以自动跟踪数据库实体的创建、修改和删除。我们将通过实际示例和代码演示，讲解如何在 Spring Boot 应用程序中实现审计功能。如何轻松地将 Auditing 注解（例如 @CreatedDate、@LastModifiedDate 和 @CreatedBy）集成到实体类中，以捕获关键元数据。

在使用数据库时，你需要跟踪谁做了什么以及何时做了什么
这意味着，例如，在创建或将数据插入数据库时​​，你需要跟踪谁做了什么以及何时做了什么；例如，当更新某些内容时，你也需要跟踪谁做了什么以及上次更新是什么时候。

深入代码之前，先解释一下在 Spring Data JPA 上下文中进行审计。
Spring Data JPA 上下文中的审计是指，允许你自动跟踪对实体所做的记录更改的功能，该实体通常是应用程序中的数据库记录，因此这对于维护审计跟踪非常有用，可以了解谁在何时进行了更改以及对数据进行了哪些具体更改。

好消息是，Spring Data JPA 已经提供了内置的审计支持，并与底层 JPA 提供程序（例如本例中的 Hibernate）的审计功能集成，因此现在让我们开始构建审计的第一个构建块。

创建日期和最后修改日期

转到 Book 实体，添加两个时间属性

为了使这两个对象可记录和可审计，我们需要做的就是添加两个注解。
spring data jpa 为我们提供了一个名为创建日期的注释：@CreatedDate，以及最后修改日期的注解：@LastModifiedDate。

现在有了实体和审计，所以现在让我们继续编写一些测试，并确保检查数据库并查看有哪些更改。

是预料之中的，所以我想向你展示这一点，因为添加其他几个注释非常重要。
所以第一件事是我们需要告诉这个实体这里有一个实体监听器，所以有一个名为 @EntityListeners 的注解。
对于这个实体监听器，我们需要传递一个或多个列表审计实体监听器，所以我们已经有一个由jpa提供的类，它叫做 AuditingEntityListener（由 org.springframework.data.jpa.domain.support 提供）。

但是注意，这还不够，你需要启用 jpa 审计，你需要做的是转到应用程序启动类，
添加注解 @EnableJpaAuditing，当我们启用 jpa 审计时，应用程序将能够跟踪我们应用程序的所有审计

您还可以将此添加到配置中包含此批注的任何类或任何配置类中。
