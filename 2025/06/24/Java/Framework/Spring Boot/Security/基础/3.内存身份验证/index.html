现在一个用户可能不够用，您的安全要求会很复杂，比如将来您可能需要有规则，所以基本上基于分配给用户的不同角色，用户应该能够访问并在系统中执行不同的操作。而且可能有不同的用户需要跟踪。

但我们只有一个应编码的 user, 所以将取消基本身份验证。
我们将讨论一种称为 InMemory Authentication(内存身份验证).

看到我们将用户详细信息存储在数据库中，UserDetailsService 正在从数据库中提取数据，所以通常情况下，你有一个数据库，其中有一个名为用户的表，🤔通常，每当需要验证用户名和密码时，都会针对该特定表进行验证。

但是现在我们还没有配置数据库，所以我们要做的是利用内存中的用户，在内存中创建用户，嗯…它们不会持久化。

这样我们就在内存中注册了两个用户。
