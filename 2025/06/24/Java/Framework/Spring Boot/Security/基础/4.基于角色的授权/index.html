任何用户都可以访问 /hello 这个端点，所以我可以使用 user1 或 admin。
在你的应用程序中会有这样的场景，比如如果你正在构建一个复杂的生产创建应用程序，通常有一个功能，你想根据分配给用户的规则来控制对应用程序不同部分的访问。

例如，你可能只希望允许管理员删除任何数据或更新任何内容，而普通用户只能查看数据。
这通常是借助注解来实现的，以便于管理，而 Spring 使它变得非常简单。

我们首先要做的是添加端点

使用 @PreAuthorize 将这些 api 的访问级别限制为具有特定角色的用户

还需要做的是，到 SecurityConfig 添加注解 @EnableMethodSecurity 来使预授权工作

403，已经过身份验证，但是访问此资源是被禁止的

@PreAuthorize 在执行方法之前运行，有后授权用于在方法执行后强制执行安全性，允许你根据方法的结果做出决定。所以也有后授权。
