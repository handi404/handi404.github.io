这是将 Spring Security 集成到项目中时发生的默认行为，这是我们尚未配置的内容

理解默认行为

首先我们访问 /hello 时，将自动进入登录表单。

在集成 Spring Security 之后的一件事，立即会看到这个特定的表单，因此端点受到保护。
这个表单我们没有构建它，这是一个内置的表单。

我们需要做的是弄清楚我们如何登录
进入控制台，看到这个密码是为开发使用而生成的，每次重启都会改变。

然后输入 username（默认 user） 和 password 即可。

想取消身份验证，该怎么办

  1. 所有端点都受到保护
  2. 默认身份验证是基于表单的身份验证，所以这是一个默认的身份验证，这意味着这种身份验证需要表单的帮助，而这个表单是一个内置表单，这是我们没有创建的。
  3. 我们被授予了访问内置表单的权限
  4. 可以访问默认用户或测试用户

可以属性配开发使用的 username password
