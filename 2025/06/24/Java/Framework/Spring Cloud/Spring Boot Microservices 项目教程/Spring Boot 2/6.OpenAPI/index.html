什么是 OpenAPI Swagger

OpenAPI 是一种用于描述 RESTful API 的标准化规范，允许开发者用 YAML 或 JSON 格式明确定义接口的结构、请求参数、返回值等信息。通过 OpenAPI 文档，可自动生成 API 文档、客户端代码或服务端框架，提升开发效率并确保一致性，其前身是 Swagger 规范。
https://www.openapis.org/what-is-openapi
OpenAPI 是一种规范，它提供了描述 HTTP api 的正式标准，正如网站中所说的那样，它主要标准化了记录 API 的实践，以便描述所有内容。例如：
这个 API 如何访问，你必须使用哪些身份验证方法来访问 API，请求主体是什么，请求参数是什么，访问 API 所需的所有信息，这些都将被记录下来，OpenAPI 提供了有关如何记录 API 的规范。

有一个名为 Swagger 的工具，它使用 OpenAPI 规范并为我们提供了一个库来记录我们的 rest api。用一个很好的类比来描述 OpenAPI 和 Swagger 之间的关系类似于 hibernate 和 jpa 的工作方式，所以在 Java 宇宙中 jpa 标准化如何使用规范与数据库交互的方式，而 hibernate 实际上是 jpa 规范的一个实现。所以同样的 OpenAPI 为我们提供了关于如何记录我们的 api 的规范，而 Swagger 为我们提供了一些工具和库，使用这些工具和库来实现 API 文档。

具体来说，当我们使用 springboot 时，我们不能直接使用 Swagger，我们必须使用一个名为 Spring Doc OpenAPI 的库来使用它。所以 Spring Doc OpenAPI 为我们提供了一个 Java 库，这样我们就可以记录我们的 rest api，它还提供了自动配置，可以自动生成 Json yaml 和 HTML格式的文档。
