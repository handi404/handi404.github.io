服务间通信重构

将 Spring Cloud Open Feign 替换为 [[RestClient]]。

原因

Spring Cloud Open Feign 实际上已经不再积极维护了。
官方简介：
如 Spring Cloud 2022.0.0 版本发布博客中所述，我们现在将 Spring Cloud OpenFeign 项目视为功能完善。我们只将添加错误修复，并可能合并一些小的社区功能 PR。我们建议迁移到 Spring Interface Clients 。
所以这实际上是在 Spring Boot 中进行服务间通信的首选方式。
除了 RestClient 之外，我们要做的就是使用 HTTP interface。在文档中 Spring 框架允许你将 HTTP Service 定义为具有 HTTP 交换方法的 Java 接口，所以这与 Spring Cloud Open Feign 非常相似。

  • RestClient - 具有流畅 API 的同步客户端。首选
