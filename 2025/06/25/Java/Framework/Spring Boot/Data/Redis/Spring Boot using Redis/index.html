什么是缓存？

现在让我们首先了解为什么需要缓存以及什么是缓存。
因此，想象一下你有一个电子商务：

在该电子商务应用程序上，客户正在访问我们的产品页面，并且该产品页面由产品端点（/product）提供服务。因此，每当客户端向应用服务器发出请求以获取产品时，我们的应用服务器都会进行数据库调用以获取产品，然后它会将其作为响应返回。想象一下我们的应用程序在促销或其他活动期间运行，会有许多个请求成千上万个请求进入应用服务器以查看产品。在这种情况下，应用程序对数据库进行了大量调用，因为数据库调用是资源密集型的，通常如果你对数据库进行了大量调用，那么你的应用程序的性能就会受到影响，为了找到这个问题的解决方案，我们可以使用一种叫做缓存的东西：

我们的应用程序将首先，每当我们第一次收到对我们的应用服务器的请求时，应用程序将从数据库中获取产品，它还将从数据库的响应保存到缓存中。因此，每当它后续请求进入应用服务器时，应用服务器将不会调用数据库来获取产品，而是首先在缓存中查找以从缓存中获取产品，并且当且仅当产品不存在于缓存中时才会调用数据库，所以这是缓存的概念。

什么是 Redis？

Redis（远程字典服务器）是一种内存数据库，最广泛地用作缓存。

  • 它还可用作数据库和消息代理
  • 设计速度极快
  • 它还能将数据存储到磁盘，因此也可用作可靠的数据库。
    如果你还没有使用像 redis 这样的缓存，那么它可以大大提高应用程序的性能，这是关于 redis 的简短介绍。
