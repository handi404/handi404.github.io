Multipart(分段、多部分)下载，提升速度和恢复能力

改进这个下载器，这个下载器代码实际上是标准代码。
但有一个问题，问题是它不支持多文件下载，或者通过将文件转换为多个块来下载文件。

怎么改善？很简单只需复制然后进行稍微修改

若调用此下载，可以看到它从四个不同的块下载，实际上它不仅仅是四个，它将支持任意数量的多个部分，例如您希望单个文件有多少个并行下载，您可以将其设置为32，最多32个，springboot确实支持它。

假设您要下载一个1GB的文件，这方法是可行的，如果下载管理器使用四个不同的部分下载文件，首先它将开始下载从 0 MB 到 250 MB 的一部分文件，同时它将开始下载第二部分，从 250 MB 到 500MB，然后同时下载 500 MB 到 750 MB 和 750 MB 到 1GB 的部分。所以这四个下载将一起进行，这样下载速度就会增加。
